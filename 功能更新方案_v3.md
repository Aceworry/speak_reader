# 功能更新方案 v3 —— 换 OCR + 朗读模式修正

## 背景
- ML Kit OCR 在设备上原生崩溃(闪退,系统提示"意外停止"),从第一版起 OCR 就没成功过。
- 决定:**彻底弃用 ML Kit**,改双方案 OCR。
- 翻译:暂缓(本轮不动)。
- 朗读:常规=第一版连续朗读+语速;听写=词组逐个播报,标点不占位。

---

## 一、OCR 换方案(核心)

### 弃用
移除 `google_mlkit_text_recognition` 依赖及 `ocr_service.dart` 现实现,
并删除 AndroidManifest 里的 ML Kit meta-data。

### 方案 A:Tesseract 离线(默认)
- 依赖 `flutter_tesseract_ocr`(不依赖 Google 服务,国产手机可用)。
- 需要**中文简体训练数据** `chi_sim.traineddata`(约 40MB)+ `eng.traineddata`。
- 放入 `assets/tessdata/`,插件从中加载。
- **构建影响**:CI 需下载 traineddata 放进 assets;APK 体积增大约 40MB+。
- 识别质量:中文可用,但排版复杂时不如云端;适合清晰印刷体。

### 方案 B:视觉大模型在线(可选,更准)
- 复用设置里的 API 配置,要求模型支持视觉(gpt-4o / qwen-vl-max / glm-4v 等)。
- 把图片转 base64,走 `chat/completions` 的 image_url 消息,提示"提取图中所有文字"。
- 新增 `vision_ocr_service.dart`。
- 设置页增加"OCR 方式"选择:离线 Tesseract / 在线视觉模型。

### 导入弹窗/流程
拍照、选图后按当前"OCR 方式"设置走 A 或 B;失败给中文提示,不崩溃。

---

## 二、朗读模式修正

TtsService 增加**两套朗读逻辑**,由 `dictationMode` 切换:

### 常规模式(回归第一版)
- 连续朗读,按句切分、句级高亮。
- **语速滑块**:直接调 `setSpeechRate`(单词本身语速),范围如 0.1~1.0。
- 不再用"词间间隔"作为常规模式调速手段。

### 听写模式(词组逐个播报)
- **英文**:按单词切。
- **中文**:按**短句/标点**切(不做词典分词)。
- **标点符号不单独占一个播报位**:切分后去除纯标点 token,标点只作为切句依据。
- 每个词组:读 → (可重复 N 遍) → 停顿(留书写)→ 下一个。
- 可整篇循环。

### 分词规则调整
`_tokenize` 分两条路径:
- 常规:切句(第一版 `_splitSentences` 逻辑)。
- 听写:英文单词 + 中文短句,**过滤掉纯标点/空白 token**。

### 高亮
- 常规:句级高亮(第一版)。
- 听写:当前词组高亮。

## 四、构建风险(如实告知)
1. **APK 体积**:含中文 traineddata,增大约 40MB+。
2. **Tesseract 插件**:`flutter_tesseract_ocr` 的 Android 原生依赖需与当前 AGP/Gradle 兼容,可能需要像之前一样在 CI 调依赖;首次构建或需 1~2 次微调。
3. **traineddata 下载**:CI 从 tesseract 官方 GitHub 下载,需网络可达(GitHub Actions 环境通常没问题)。
4. 若离线识别质量不满意,可切到方案 B(视觉大模型)获得更好效果。

---

## 五、实施顺序
1. pubspec 换依赖 + assets;删 ML Kit manifest 项
2. 重写 ocr_service(Tesseract)+ 新增 vision_ocr_service
3. settings 加 OCR 方式;home 按方式调用
4. TtsService 常规/听写双逻辑(听写过滤标点、中文按短句)
5. reader_page/settings_page UI 调整
6. CI 加 traineddata 下载
7. 推送出 APK,真机验证 OCR 与两种朗读模式

## 六、验证重点
- OCR 是否**不再闪退**、能出文字(先试离线,再试在线视觉)
- 常规模式:连续朗读 + 语速滑块正常
- 听写模式:词组逐个、标点不占位、中文按短句、可重复/停顿/循环
