# 功能更新方案 v2 —— OCR 修复 + 翻译 + 逐词听写 + 词间间隔

## 需求
1. 修复 OCR(图片/PDF)崩溃退出
2. 新增可自设 API 的翻译(OpenAI 兼容接口)
3. 英文逐词朗读 + 反复阅读(报听写形式,逐词自动循环)
4. 语速调节改为「词与词间隔时间」,单词本身自然速度读

---

## 一、OCR 崩溃修复(务实说明)
"崩溃退出"多为**原生层崩溃**,Dart 的 try-catch 抓不住。按最常见原因加固:
- **限制图片尺寸**:`image_picker` 取图时设 `maxWidth/maxHeight`(如 2000px),避免超大图让 ML Kit 原生 OOM 崩溃(最常见原因)。
- **OCR 前压缩兜底 + 错误捕获**:识别失败给中文提示而非闪退。
- **PDF**:当前用 syncfusion 提取文本层。扫描版/图片版 PDF 无文本层 → 提取为空,给出明确提示"该 PDF 可能是扫描件,请改用拍照导入",不再让用户以为是 bug。
- **诚实前提**:若加固后仍崩,需你在真机反馈崩溃场景(哪种导入、什么图/文件),或提供 logcat 才能进一步定位。

---

## 二、TTS 引擎重构(核心,一套机制满足需求 3+4)
把朗读从"逐句"改为"**逐词(token)**":
- 文本 → token 序列:
  - **英文**:按空格/标点切成单词
  - **中文**:按标点切成短句(中文无空格,逐字朗读不自然)
- 逐个 `speak(token)`,**token 之间插入可调延迟 `wordGapMillis`**。
- `setSpeechRate` 固定为自然值(如 0.5),不再作为主要调速手段。
- **"语速"滑块含义变更** → "词间间隔"滑块(0.0s ~ 2.0s),值越大读得越慢(停顿越长),单词本身语速不变。

### 高亮
从句级高亮改为 **token 级高亮**(当前朗读的词高亮 + 自动滚动)。

### 听写模式(dictation)
在逐词基础上增加:
- `repeatCount`:每个词连读 N 遍(默认 2)
- `dictationGapMillis`:词间更长停顿(默认 2s,留给听写者书写)
- 读完整篇可 `loop` 循环重来
- UI:阅读页顶部一个「听写模式」开关

### TtsService 新增字段/方法
- `wordGapMillis` / `repeatCount` / `dictationGapMillis` / `dictationMode` / `loop`
- token 拆分 `_tokenize(text)`(区分中英文)
- 逐 token 播放循环(支持暂停/继续/跳词/从某词开始)

## 三、翻译功能(OpenAI 兼容)
### 新增 `TranslationService`
- 用 `http` 包 POST `{baseURL}/chat/completions`
- 请求头 `Authorization: Bearer {key}`
- body:`{model, messages:[{role:system,系统提示"翻译为中文"},{role:user,原文}]}`
- 解析 `choices[0].message.content` 得译文
- 兼容 OpenAI / DeepSeek / 通义 / 智谱 / Kimi 等(它们都提供 OpenAI 兼容端点)

### 配置持久化 `SettingsService`(shared_preferences)
存:`baseURL`、`apiKey`、`model`,以及词间间隔/听写重复次数等默认值。

### UI
- 新增**设置页** `settings_page.dart`:填写 API 地址/密钥/模型,测试连接。
- 阅读页新增**「翻译」按钮**:翻译当前全文(或当前句),弹出/下方显示中文译文。
