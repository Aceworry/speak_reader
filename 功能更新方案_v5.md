# 功能更新方案 v5 —— v2.1.0

在 v2.0.0(音色选择+音频导出)基础上,按 6 项需求继续增强。受 Android 平台
硬约束,其中"集成 TTS 声音文件""调用厂商 OCR/翻译"改为可行的替代方案。

## 阶段 A(已实现,v2.1.0 首批)

### ② 导出按当前朗读模式执行
`audio_export_service` 重构:`exportDocument` 接收 `dictation` + 各参数。
- 常规模式:整段连读(分块合成拼接),用常规语速。
- 听写模式:逐词组合成 → 每词重复 N 遍(复制该词 PCM)→ 词间插入停顿秒数对应的
  **静音 PCM**(全 0 字节,匹配采样率/声道/位深)→ 拼成单个 WAV。用听写语速。
音色/音调始终按当前设置。文件名带"_听写"后缀区分。

### ③ 听写单词语速调节
新增独立参数 `dictationRate`(默认 0.4),与常规 `speechRate` 分开。设置页 +
阅读页听写区各一个滑块;`setDictationRate` 播放中即时生效;导出也用它。

### ④ 停顿 10 秒 / 重复 10 次
书写停顿滑块 6.0s→**10.0s**;每词重复 5→**10**。设置页与阅读页同步放宽。

### ⑤ 导出目录可选
新增 `customOutputDir`,用 `file_picker` 的 `getDirectoryPath()` 选目录。
选定后先写测试文件校验可写(`isDirWritable`),不可写则提示并回退应用私有目录
(不引入 MANAGE_EXTERNAL_STORAGE 重权限)。可"恢复默认"。

### ① TTS 声音「引导跳转安装」
加 `android_intent_plus ^5.0.0`。检测目标语言无声音时,设置页/音色面板显示
"下载更多声音",点击发 `android.speech.tts.engine.INSTALL_TTS_DATA` intent 跳
系统安装页;失败回退 `market://` Google TTS 页。
⚠️ **Android 不允许第三方 App 把声音包打进 APK 给系统引擎加载**(引擎只认自己
目录的数据),引导安装是同类 App 的标准做法。

## 阶段 B(计划中,独立提交)

### ⑥ PaddleOCR 离线识别(免 API key)
`OcrMode` 加 `paddleOffline`;加 `fast_paddle_ocr` + NCNN 模型进 assets(CI 下载);
新增 `paddle_ocr_service.dart`。
⚠️ 风险:该插件主打相机实时流,静态图片文件接口待验证;原生 NCNN+CI 打包模型
历史上易反复(Tesseract 曾编译失败),故独立提交、多半需 1~2 次 CI 调试。

### ⑥ 厂商 OCR/翻译 —— 技术不可行,已排除
三星/小米/VIVO/OPPO 的 OCR/翻译是**深度集成到系统固件的私有能力,无开放 API
给第三方 App 调用**(实测证实无法移植/调用)。翻译无免 key 离线方案(ML Kit 离线
翻译依赖 Google 服务、国产机不稳),故**翻译维持在线 API**。

## 涉及文件(阶段 A)
| 文件 | 改动 |
|------|------|
| `pubspec.yaml` | 版本 2.1.0;加 `android_intent_plus ^5.0.0` |
| `settings_service.dart` | 加 `dictationRate`、`customOutputDir` 字段 + 持久化 |
| `tts_service.dart` | `dictationRate` + `_effectiveRate`;`setDictationRate`;`synthToFile(rate:)`;`tokenizeForDictation`;`hasVoiceFor`;`openInstallVoiceData`(INSTALL_TTS_DATA intent) |
| `audio_export_service.dart` | 按模式导出(听写逐词重复+静音停顿);`customDir`+可写校验+回退;静音 PCM 生成 |
| `settings_page.dart` | 听写语速滑块、停顿→10s、重复→10、目录选择器+恢复默认、下载声音按钮 |
| `reader_page.dart` | 听写语速滑块、范围放宽、按模式导出、音色面板下载声音入口 |

## 构建注意
- `android_intent_plus`:`AndroidIntent(...)` **非 const 构造**,必须用 `final`;
  ACTION_VIEW 用完整字符串 `android.intent.action.VIEW`。
- `file_picker` 已在依赖内,复用其 `getDirectoryPath()`。
