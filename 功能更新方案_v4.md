# 功能更新方案 v4 —— 音色选择 + 后台音频录播导出(v2.0.0)

## 版本
`pubspec.yaml`:`1.0.0+1` → **`2.0.0+1`**

## 新增能力

### 一、音色选择
在「设置」与阅读页均可切换音色,三条维度:

1. **语言/口音**:枚举系统 TTS 引擎的 `getLanguages`,提供中文(简体/繁体/香港)、
   美式英语(en-US)、英式英语(en-GB)、澳/印/加英语、日/韩/法/德等下拉。
   口音在各引擎稳定可靠。
2. **具体声音(男/女等)**:枚举 `getVoices`,按当前语言过滤,列出可选声音。
   Android 各引擎对"性别"暴露方式不统一,故采用**声音名启发式**匹配男/女声
   (维护 female/male 关键词表),匹配不到回退引擎默认声音并给出提示。
3. **音调 pitch(0.5~2.0)**:标准 1.0;童声通过提高音调(默认 1.7)模拟。

**快捷预设**(一键):系统默认、中文·女、中文·男、美式·女、美式·男、
英式·女、英式·男、童声(高音调)。

试听按钮:用当前音色朗读一句示例文本。

### 二、后台自动录播 + 音频文件导出
- **手动导出**:阅读页顶部"导出音频"按钮,带进度条,生成后弹窗可"分享/打开"。
- **自动导出**:设置里开启"朗读时自动生成音频文件",进入阅读页 / 编辑保存后
  在后台静默合成一份(固定文件名,覆盖式,避免堆积),完成后 SnackBar 提示可分享。
- **格式**:WAV。Android 原生 `synthesizeToFile` 只产出 WAV/PCM;MP3 需 ffmpeg 类
  转码依赖会拖累本项目脆弱的 CI 构建,故 v2.0.0 只做 WAV(枚举保留扩展余地)。
- **长文本**:按 ≤2000 字分块合成到临时 WAV,再解析各块 PCM `data` 区**拼接为单个 WAV**
  (校验采样率/声道/位深一致),最后清理临时文件。
- **输出目录**:`<应用外部存储>/speak_reader_audio/`(应用作用域存储,**无需运行时权限**,
  可被系统文件管理器访问)。
- **文件管理**:音色面板里"查看已生成的音频"底部弹窗,可分享 / 删除。

## 构建注意(如实告知)
1. **share_plus 版本**:锁 `^10.1.0`(安全区间 10.1.0~11.x)。<10.0.1 依赖 `web ^0.5.0`,
   与 `file_picker ^8.1.2`(要 `web ^1.0.0`)冲突致 pub 解析失败;12.0.0+ 强制
   AGP≥8.12/Kotlin 2.2,与当前 CI(compileSdk 36、关混淆)冲突。
   10.x/11.x 用 `web ^1.0.0`、AGP 要求温和,仍保留旧 API
   `Share.shareXFiles([XFile], text:)` → `ShareResult.status`。
2. **音频写入无需权限**:走 `getExternalStorageDirectory()` 应用私有目录,插件用
   ParcelFileDescriptor/File 直接写,不触发存储权限弹窗。
3. CI 已把各插件 compileSdk 统一提升到 36,`path_provider`/`share_plus` 兼容;
   自定义 lib 文件由 `git checkout -- lib` 恢复,新增文件会保留。

- 自动导出:开启后进入阅读页后台生成,SnackBar 提示;长文本能拼成单个完整 WAV。
