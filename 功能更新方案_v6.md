# 功能更新方案 v6 —— v2.2.0

在 v2.1.0 基础上,按用户三项需求增强:①听写每词重复之间的时间间隔、
②修复"下载更多声音"打不开系统语音数据安装页、③离线翻译 + 离线文字识别。

## ① 听写:每词重复之间的时间间隔(可配置)
此前重复间隔写死为 `_repeatGapSeconds = 0.6`。现改为可配置:
- `settings_service`:新增 `repeatGapSeconds`(默认 0.6,范围 0~5s)+ 持久化键 `cfg_repeat_gap`。
- `tts_service`:`_repeatGapSeconds` → 可写字段 `repeatGapSeconds`;`_runLoop` 重复遍之间用它。
- `audio_export_service`:`exportDocument`/`_exportDictation` 新增 `repeatGapSeconds`;
  **重复遍之间**插"重复间隔"静音、**词与词之间**插"书写停顿"静音(此前二者混用一个静音,已分开)。
- `settings_page` + `reader_page`:各加一个"重复间隔"滑块,听写且重复≥2遍时显示/生效。

## ② 修复"下载更多声音"跳转
根因:Android 11+ 包可见性 + 部分引擎不响应 `INSTALL_TTS_DATA`,原实现仅两级回退且未在
manifest 声明查询目标,故 `ActivityNotFoundException`。
- `tts_service.openInstallVoiceData`:改为四级回退,任一成功即返回:
  1. `INSTALL_TTS_DATA`(引擎安装语音数据页)
  2. `com.android.settings.TTS_SETTINGS`(系统 TTS 设置页)
  3. `market://…com.google.android.tts`(应用商店)
  4. `https://play.google.com/…`(浏览器兜底)
  全失败才抛错并提示手动路径。新增 `_tryLaunch` 辅助。
- `AndroidManifest.xml`:`<queries>` 补声明 INSTALL_TTS_DATA、TTS_SETTINGS、VIEW(market/https),
  否则 Android 11+ 查不到目标、intent 直接失败。

## ③ 离线翻译 + 离线文字识别

### 离线 OCR —— ML Kit Text Recognition(完全本地,无需 API)
- 新增 `mlkit_ocr_service.dart`:`google_mlkit_text_recognition`,用中文识别器
  (`TextRecognitionScript.chinese`,可识别中英文混排)。
- `home_page._runOcr`:按 `settings.ocrMode` 分支——`offline`→ML Kit(无需 key,
  **try-catch 包裹,失败/无 GMS 自动回退在线视觉模型**),`vision`→在线视觉模型。
- 注:`OcrMode.offline` 枚举 v2.1.0 就已存在但从未接线,本次才真正实现。
- ⚠️ **原定 Tesseract 方案已废弃**:`flutter_tesseract_ocr` 已停更,其
  `android/build.gradle` 用了下线的 `jcenter()`,与 CI 的 AGP 9 不兼容,构建直接失败。
  改用同属 ML Kit 生态的 text recognition,与离线翻译共用依赖,零额外构建风险。

### 离线翻译 —— ML Kit + 在线回退
- 新增 `offline_translation_service.dart`:`google_mlkit_language_id` 识别源语言 +
  `OnDeviceTranslator` 翻译。目标语中文;源为中文则译英文。首次用 `ModelManager`
  按需下载语言模型(仅 WiFi),之后离线。
- `settings_service`:新增 `preferOfflineTranslation`(默认开)+ 键 `cfg_prefer_offline_tr`。
- `reader_page._translate`:优先离线,任何异常(无 GMS / 模型缺失等)**自动回退在线 API**
  并 toast 告知;若无在线兜底则把离线错误抛给用户。
- `settings_page`:新增"文字识别方式"单选(离线/在线)+"优先离线翻译"开关。

## 依赖 / 构建
- `pubspec.yaml`:版本 2.2.0;加 `google_mlkit_text_recognition ^0.13.0`、
  `google_mlkit_translation ^0.13.0`、`google_mlkit_language_id ^0.13.0`。
- CI `build-apk.yml`:无需额外资源(ML Kit 模型运行时下载)。现有 compileSdk36 +
  关混淆补丁对 ML Kit 继续适用。

## ⚠️ 风险(用户已选一次性提交,记忆已有踩坑记录)
- **Tesseract 方案在 CI 首次构建即失败**(jcenter/AGP9 不兼容),已改 ML Kit OCR。
- **ML Kit 依赖 Google Play 服务**:无 GMS 的国产机上离线 OCR/翻译不可用——
  OCR 已 try-catch 回退在线视觉模型,翻译已回退在线 API,均有兜底。
- ML Kit 可能要求 minSdkVersion 21+;若 CI 报 minSdk 相关错,需在
  `android/app/build.gradle.kts` 提升 minSdk。
